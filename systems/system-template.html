Modular Inventory Framework (Example)

A drag-and-drop replicated inventory component for UE5.

StatusCompleted (V 1.0)
ArchitectureData-Driven / Component Based
AvailabilityOpen Source

System Architecture

I built this framework because I was tired of rewriting inventory logic for every prototype. This system is completely decoupled from the player character—it is a single UActorComponent that can be attached to players, chests, or vehicles.

// Example: Adding an item server-side is highly secure bool UInventoryComponent::TryAddItem(UItemData* Item, int32 Quantity) { if (!GetOwner()->HasAuthority()) return false; if (CurrentWeight + (Item->Weight * Quantity) > MaxWeight) { Client_OnInventoryFull(); return false; } // Process Add Logic... return true; }

Core Features

  • 100% Server Authoritative: Prevents client-side item duplication and weight spoofing.
  • Weight & Slot Systems: Can be configured to use grid-based slots or simple weight limits.
  • Event Dispatchers: UI automatically updates by listening to OnInventoryChanged delegates, keeping UI logic separated from backend logic.

How to integrate it

To use this system in a new project, a developer simply needs to:

  1. Attach UInventoryComponent to any Actor.
  2. Create Data Assets inheriting from UItemData.
  3. Bind the UI to the Component's broadcast events.
